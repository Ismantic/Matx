# 序章：MC编译器系统概述

## 要解决的问题

MC编译器的目标很简单：**将Python代码编译成等价的C++代码，在保持Python语义的同时获得C++的执行性能**。

考虑这个Python函数：
```python
def add_numbers(a: int, b: int) -> int:
    return a + b
```

我们希望将它编译成这样的C++代码：
```cpp
int64_t add_numbers(int64_t a, int64_t b) {
    return a + b;
}
```

看起来简单，但实际上这涉及到两个根本不同的编程范式的转换。

## 核心挑战

### 1. 类型系统的差异
- **Python**：动态类型，运行时类型检查，一切皆对象
- **C++**：静态类型，编译时类型确定，基础类型vs对象类型

### 2. 内存管理的差异  
- **Python**：自动垃圾回收，对象引用语义
- **C++**：手动内存管理，需要显式的生命周期控制

### 3. 数据结构的差异
- **Python**：`list`可存储任意类型，`dict`的键可以是任意可哈希类型
- **C++**：`std::vector<T>`需要确定类型，`std::map<K,V>`需要确定键值类型

### 4. 运行时特性的差异
- **Python**：支持`isinstance()`、动态属性访问、反射
- **C++**：编译时确定所有类型和结构

## 基本思路

MC编译器的解决方案是构建一个**运行时系统**，在C++中模拟Python的动态特性：

### 1. 统一的对象模型
```cpp
class object_t {
    int32_t t_;                    // 运行时类型索引
    std::atomic<int32_t> count_;   // 引用计数
    // 所有对象都继承自这个基类，提供统一的内存管理
};
```

### 2. 类型擦除的值系统
```cpp
struct Value {
    union {
        int64_t v_int;
        double v_float;
        void* v_pointer;
    } u;
    int32_t t;  // 类型标签
};
```
通过联合体+类型标签，可以在C++中实现类似Python的"一切皆对象"。

### 3. Python数据结构的C++实现
```cpp
class List : public object_r {
    std::vector<McValue> data_;  // 可以存储任意类型
};

class Dict : public object_r {
    std::unordered_map<McValue, McValue> data_;  // 任意键值对
};
```

### 4. 编译流程设计
```
Python代码 → Python AST → MC AST → C++代码
```

## 编译流程

以这个Python函数为例：
```python
def process_list(items: list) -> int:
    total = 0
    for item in items:
        if isinstance(item, int):
            total += item
    return total
```

### 第1步：解析为AST
将Python代码解析为抽象语法树，识别函数、循环、条件等结构。

### 第2步：类型推断和IR转换
```cpp
PrimFunc process_list = PrimFunc(
    params = {PrimVar("items", ListType())},
    body = SeqStmt({
        AllocaVarStmt("total", IntType(), IntImm(0)),
        ForStmt(...),
        ReturnStmt(...)
    }),
    return_type = IntType()
);
```

### 第3步：代码生成
```cpp
int64_t process_list(List items) {
    int64_t total = 0;
    for (const auto& item : items) {
        if (item.Is<int64_t>()) {
            total += item.As<int64_t>();
        }
    }
    return total;
}
```

## 系统架构

MC编译器包含以下核心组件：

1. **Object**：统一的对象模型，解决内存管理和类型系统问题
2. **Container**：Python数据结构的C++实现（List、Dict、Set等）
3. **AST**：抽象语法树表示，支持表达式、语句、函数等
4. **Visitor**：AST遍历和变换框架，实现代码生成和优化
5. **Value**：统一的值表示系统，支持类型擦除和安全转换
6. **Function**：函数注册和调用机制，支持任意C++函数的统一调用
7. **FFI**：Python和C++的互操作层，实现无缝的双向调用

## 关键特性

### 1. 语义保持
生成的C++代码在行为上与原Python代码完全一致，包括异常处理、类型检查等。

### 2. 性能优化
- 引用计数替代垃圾回收
- 类型索引替代动态类型查找
- 零拷贝的数据传递

### 3. 渐进式编译
支持混合编程：可以只编译性能关键的部分，其他部分继续使用Python。

### 4. 类型安全
虽然支持动态类型，但通过运行时类型检查保证类型安全。

## 性能效果

实际测试中，编译后的代码通常能获得10-100倍的性能提升，具体取决于代码的特点。
